## 启动后切换版本的方式

### 方式 1：浏览器地址栏直接访问

在浏览器中访问以下地址即可切换：

**切换到 v2：**
```
http://localhost:3000/api/admin/switch-frontend
```
在浏览器控制台（F12）执行：
```javascript
fetch('/api/admin/switch-frontend', {method:'POST', headers:{'Content-Type':'application/json'}, body:JSON.stringify({version:'v2'})}).then(r=>r.json()).then(console.log)
```

**切换到 v1：**
```javascript
fetch('/api/admin/switch-frontend', {method:'POST', headers:{'Content-Type':'application/json'}, body:JSON.stringify({version:'v1'})}).then(r=>r.json()).then(console.log)
```

### 方式 2：CMD 中切换

```cmd
curl -X POST http://localhost:3000/api/admin/switch-frontend -H "Content-Type: application/json" -d "{\"version\":\"v1\"}"
```

```cmd
curl -X POST http://localhost:3000/api/admin/switch-frontend -H "Content-Type: application/json" -d "{\"version\":\"v2\"}"
```

### 方式 3：管理页面中切换

在管理页面 AdminPage.vue 中已有版本切换入口，点击即可切换。

---

## 工作流程

```
启动时 → 选择 v1 或 v2
运行中 → 浏览器控制台 / CMD / 管理页面 随时切换
切换后 → 刷新页面即可看到新版本
```

## `scripts/snapshot-all.js`快照使用方法

### 创建快照（在修改代码前执行）

```cmd
npm run snapshot
```

### 查看所有快照

```cmd
npm run snapshot:list
```

### 恢复快照

```cmd
node scripts/snapshot-all.js restore backup_2026-06-01_14-30-00
```

### 删除指定快照

```cmd
node scripts/snapshot-all.js delete backup_2026-06-01_14-30-00
```

## 功能对比

| 功能 | 原数据库快照 | 新全量快照 |
|---|---|---|
| 数据库 `data/app.db` | 不含（排除 data 目录） | 不含（排除 data 目录） |
| 前端源代码 `src/` | 不含 | 包含 |
| 测试文件 `tests/` | 不含 | 包含 |
| 配置文件 | 不含 | 包含 |
| `node_modules` | 不含 | 排除（体积过大） |
| `dist` 构建产物 | 不含 | 排除（可重新构建） |
| 恢复前自动备份 | 无 | 有（`pre-restore_*`） |
| 文件清单 | 无 | 有（`.snapshot-manifest.json`） |
| 最大保留数 | 20 | 10 |

---

## 建议工作流

```
修改代码前 → npm run snapshot        创建快照
修改代码    → 进行修改
出现问题    → node scripts/snapshot-all.js restore <快照名称>
正常运行    → npm run build           重新构建
```

建议在每次重大修改前执行一次快照，这样随时可以回退。
