## 系统简介

每日数据填报系统是一款团队协作工具，用于收集和管理人员的日常数据填报。系统支持：

- **多模板管理**: 创建和管理多个表单模板
- **多人协作**: 按责任人分配填报任务
- **数据继承**: 自动继承上一条填报记录，减少重复录入
- **字段规则**: 灵活的字段校验和联动规则
- **数据分析**: 多维度统计分析和交叉报表
- **数据导出**: 支持 CSV / Excel 导出
- **审计追踪**: 完整的操作日志记录

## 快速入门

### 1. 启动系统

```bash
# 启动后端服务
npm start
```

浏览器访问 `http://localhost:3000`。

### 2. 首次使用

1. 进入「填报」页面，选择模板和填报人
2. 展开记录卡片，填写各字段
3. 点击「提交今日填报」保存数据

### 3. 管理员登录

1. 点击导航栏进入「管理」或「统计」页面
2. 输入默认密码 `1234`（建议首次登录后修改）
3. 进入管理后台

## 填报页面

填报页面是系统的核心功能，用于日常数据录入。

### 页面布局

| 区域 | 说明 |
|------|------|
| **顶部栏** | 系统标题、日期选择、网络连接状态 |
| **侧边栏** | 模板选择、用户选择、日期筛选 |
| **主区域** | 记录卡片列表、表单编辑区 |

### 基本操作流程

1. **选择模板**: 从下拉列表中选择要填报的模板
2. **选择用户**: 选择当前填报人（影响数据归属和筛选字段匹配）
3. **查看记录**: 系统自动显示该模板下的所有记录卡片
4. **填写数据**:
   - 点击卡片展开表单
   - 填写各字段（必填字段标有 `*`）
   - 继承标记显示数据来源（今日/历史/基础）
5. **提交**: 点击「提交今日填报」保存所有修改

### 快捷操作

| 操作 | 说明 |
|------|------|
| **快速填写** | 弹出快捷填写对话框，快速填入常用数据 |
| **新增行** | 在模板中添加新的基础行 |
| **批量新增** | 一次性添加多条记录 |
| **继承昨日** | 将整个昨日的填报数据继承到今天 |
| **继承基础** | 将模板的基础数据继承到今天 |
| **重置行** | 重置单条记录为基础数据 |

### 状态标签

| 标签 | 含义 |
|------|------|
| 已提交 | 该记录已提交，不可修改 |
| 填报中 | 该记录有未提交的修改 |
| 未填 | 该记录尚未开始填写 |

### 进度条

每张卡片底部显示填写进度：`已填字段 / 总字段`。

## 历史页面

历史页面用于查看和浏览过去的填报记录。

### 两种视图

| 视图 | 触发条件 | 说明 |
|------|----------|------|
| **按日期聚合** | 未选择具体用户 | 显示所有用户的填报概况，按日期分组 |
| **按用户列表** | 选择了具体用户 | 显示该用户在各个日期的填报记录 |

### 查看记录详情

1. 在历史列表中点击某条记录
2. 弹出详情窗口显示完整的填报数据
3. 可从此处直接导出该日期的数据

### 数据导出

在历史页面的用户视图中，每条记录旁有导出链接，可单独导出某一天的数据。

## 统计页面

统计页面提供多维度的数据分析，需密码访问。

### 概览 Tab

显示系统整体填报情况：

| 指标 | 说明 |
|------|------|
| **总记录数** | 当前模板下的总行数 |
| **已填数** | 有提交数据的记录数 |
| **未填数** | 尚未填报的记录数 |
| **填报率** | 已填数 / 总记录数的百分比 |
| **趋势图** | 近 N 天的填报趋势 |
| **成员完成率** | 每个成员的填报完成率排行 |

### 数据分析 Tab

| 功能 | 说明 |
|------|------|
| **数值统计** | 对数值型字段自动计算：求和、平均值、最大值、最小值 |
| **数据明细** | 按日期和用户查看原始填报数据 |

### 交叉分析 Tab

强大的透视表功能：

1. **配置行维度**: 选择作为行分组依据的字段（可多选）
2. **配置列维度**: 选择作为列分组依据的字段
3. **配置统计指标**: 选择统计函数（计数/求和/平均/最大/最小/占比）
4. **生成表格**: 点击生成按钮查看交叉分析结果

管理页面用于系统配置和数据管理，需密码访问。

### 概览 Tab

### 模板管理 Tab

| 功能 | 说明 |

编辑器包含三个子面板：

1. **字段管理**: 添加/编辑/删除字段，设置类型（文本/数字/下拉/日期等）、必填、可编辑等属性
2. **基础数据**: 设置模板的默认行数据
3. **字段规则**: 配置字段间的联动和校验规则

### 数据查看 Tab

| 功能 | 说明 |
|------|------|
| **数据筛选** | 按日期、用户筛选 |
| **编辑数据** | 修改已有的填报数据 |

### 数据导入 Tab

支持从 JSON 文件导入模板配置：

1. 点击「选择文件」上传 JSON 文件
2. 预览导入内容
3. 配置字段映射（如有重名字段会自动重命名）
4. 确认导入

### 数据导出 Tab

| 格式 | 说明 |
|------|------|
| **CSV** | 通用表格格式，适合文本编辑器打开 |
| **Excel (.xlsx)** | Microsoft Excel 格式，自动列宽适配 |

导出选项：
- 按模板筛选
- 按日期范围筛选
- 按用户筛选

### 审计日志 Tab

查看系统操作记录：

| 信息 | 说明 |
|------|------|
| **时间** | 操作发生的时间 |
| **操作** | 具体做了什么（登录/创建模板/提交数据等） |
| **分类** | 操作所属类别（认证/模板/数据/成员/系统） |
| **用户** | 操作用户 |
| **IP** | 操作来源 IP |

支持按分类和操作类型筛选，最多可查看 2000 条记录。

### 设置 Tab

| 功能 | 说明 |
|------|------|
| **成员管理** | 添加/删除填报成员 |
| **修改密码** | 更改管理员密码 |

| **重置数据** | 清空所有模板、提交和成员数据（危险操作） |

## 常见问题

### Q: 填报数据继承是怎么工作的？

A: 系统采用三级继承机制：
1. **优先使用今日已提交的数据**
2. 如果今日为空，使用**最近一次历史填报**的数据
3. 如果历史也为空，使用**模板基础数据**

继承按**整行**进行，不会混合不同日期的字段，确保数据完整性。

### Q: 如何配置字段规则？

A: 在模板编辑器的「字段规则」面板中配置：
- **条件**: 选择字段、操作符（等于/包含/大于等）、值
- **动作**: 选择动作类型（必填/禁止/复制/校验匹配等）
- 规则按顺序执行，可同时配置多条

### Q: 忘记管理员密码怎么办？

A: 可以通过以下方式重置：
1. 如果有数据库访问权限，直接在 `system` 表中修改 `adminPwd` 的值
2. 联系系统管理员或通过数据库恢复默认密码

### Q: 数据丢失了怎么恢复？

A: 系统有多种恢复机制：
1. **项目快照**: `node scripts/snapshot-all.js restore <快照名称>`
2. **自动备份**: `data/backups/auto/` 目录保留最近 7 天的数据库备份
3. **手动备份**: `data/backups/manual/` 目录

### Q: 如何在手机上使用？

A: 系统已适配移动端布局。在手机浏览器中访问系统地址即可，界面会自动切换为移动端布局（顶部导航 + 底部标签栏）。

### Q: 导出文件的编码格式是什么？

A: CSV 文件使用 UTF-8 BOM 编码，确保在 Excel 中打开时中文正常显示。Excel 文件为标准 .xlsx 格式。

### Q: 最多支持多少行数据？

A: 前端渲染限制为 1000 行（`MAX_RENDER_ROWS`），导出功能限制为 50000 行。如需处理更多数据，建议使用导出后在 Excel 中分析。
